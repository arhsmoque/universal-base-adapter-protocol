> For small components, prefer `METADATA.yml` / `component.manifest.yml`. Use this template when review depth is needed.

# Housekeeping Checklist

## Keep
- [ ] final source files
- [ ] tests
- [ ] final config/rules
- [ ] active adapters
- [ ] provenance logs
- [ ] decision records
- [ ] verification outputs
- [ ] user-facing artifacts

## Archive / Quarantine
- [ ] original source snapshots used for porting
- [ ] rejected candidates worth remembering
- [ ] comparison notes
- [ ] failed spikes with useful negative evidence

## Delete
- [ ] scratch files
- [ ] temp ports
- [ ] generated candidates not selected
- [ ] duplicate renamed copies
- [ ] stale wrappers
- [ ] unused downloads
- [ ] unreferenced logs
- [ ] partial artifacts that pollute filename discovery

## Search-Pollution Check
- [ ] searched temp/scratch/candidate filenames
- [ ] searched duplicate module names
- [ ] searched TODO markers created during salvage
- [ ] final docs point to canonical files only
- [ ] provenance moved to archive, not active source paths
- [ ] artifact index or decision log updated
