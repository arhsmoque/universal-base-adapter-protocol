# Universal Design Anti-Patterns — Quick Reference

Canonical authority: `UNIVERSAL_BASE_ADAPTER_PROTOCOL.md` section 19.

Use this file only as a quick pointer for agents and reviewers. Do not maintain a divergent anti-pattern list here.

Most important anti-patterns to check:

- adapter-first architecture
- API mirror tools
- creative rewrite after reading source
- rules buried in base
- one script owns everything
- unstructured raw output
- hidden mutation
- cleanup as optional
- no rejection log
- no primitive escape hatch
- completion without verification
